## 项目

### 1.**大宗商品指数管理系统**

- **技术架构**：Spark + Hadoop + Hive + Tidb + BatchWorks

- **行业**：金融/大宗商品

  核心亮点

  - 整合多源数据（交易所、第三方、行业报告），使用 **Spark** 作为核心计算引擎，处理千万级指标数据。
  - 优化复杂Hive SQL任务为Spark代码，解决迭代计算性能瓶颈，任务执行时间缩短45%。
  - 构建分层数仓（ODS→DWD→DWS→ADS），支持多维指标分析（价格、库存、产量）。

  代表性技术点

  - **Spark SQL** 动态分区裁剪与并行度调优。
  - 利用 **Spark Core** 处理数据倾斜（拆分热点值、负载均衡）。

### **2. 美柚离线数仓平台**

- **技术架构**：Spark + Hive + Flume + Kafka + Dolphinscheduler

- **行业**：互联网/女性健康

  核心亮点

  - 基于 **Spark on Hive** 开发ETL流程，日均处理50亿+用户行为数据。
  - 使用 **Spark SQL** 实现复杂指标（留存率、复购率），优化JOIN操作与Shuffle性能。
  - 通过Spark参数调优（内存管理、并行度），解决小文件合并与资源争夺问题。

  代表性技术点

  - **Spark调优**：Map端聚合、内存溢出控制（`spark.memory.fraction`）。
  - 利用 **Spark Streaming** 补充实时计算场景（T+1与实时结合）。



### **3. 商用车大数据平台**

- **技术架构**：Spark + Hadoop + Hive + Flume + HBase

- **行业**：汽车/物流

  核心亮点

  - 构建五层数仓（ODS→DWD→DWS→DIM→APP），使用 **Spark** 处理车辆轨迹数据（日均PB级）。
  - 开发基于 **Spark MLlib** 的DBSCAN算法，实现车辆聚集地自动识别（准确率90%）。
  - 优化Spark任务（动态资源分配、Shuffle调优），集群资源消耗降低30%。

  代表性技术点

  - **Spark GraphX** 优化路径匹配算法（道路寻车准确率提升50%）。
  - **Spark SQL** 物化视图加速多维分析查询。

### **4. 丽兹行实时数仓项目（文档4：魏云龙）**

- **技术架构**：Flink + Kafka + HBase + Doris + FlinkCDC

- **行业**：房地产/高净值客户服务

  核心亮点

  - 构建房产交易实时数仓，**Flink** 处理实时业务数据（如带看量、成交量）和日志流（顾问行为）。
  - 使用 **Flink CDC** 实时捕获MySQL业务库变更（楼盘字典、客户跟进记录），实现端到端秒级延迟。
  - 开发风险预警模块（如顾问消息超时回复），通过 **Flink窗口计算**（滑动窗口+事件时间）动态统计指标。
  - 结果写入 **Doris** 支持多维即席查询，替代原ClickHouse方案，查询性能提升50%。

  代表性技术点

  - **Flink双流JOIN** 关联实时交易数据与维度数据（HBase存储）。
  - **状态TTL** 自动清理过期状态数据（如7天未活跃客户）。
